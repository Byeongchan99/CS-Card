# CS-Card 저장소 작업 규칙

CS 면접 준비용 질문 카드 저장소. 카드는 claude.ai 프로젝트의 공부 세션에서
생성되어 이 repo에 커밋되고, Quartz 기반 GitHub Pages 사이트로 복습에 사용된다.

## 저장소 구조
- `cards/` — 질문 카드 (.md). 카드 하나 = 질문 하나.
- `index.md` — 카드 인덱스. 정비 시 자동 재생성되며, claude.ai 프로젝트 지식에
  연동되어 공부 세션의 중복 출제 방지에 사용된다.
- `quartz/`, `quartz.config.ts`, `quartz.layout.ts` — Quartz 4 사이트 소스(vendored).
- `content/` — 사이트 전용 페이지: 홈 `index.md`, 랜덤 퀴즈 `quiz.md`.
  카드는 빌드 시 `scripts/sync-content.mjs`가 `content/cards/`로 복사하면서
  상위 질문 링크와 "연관 카드" 섹션을 주입한다(원본 카드는 수정하지 않음).
  `content/cards/`와 `content/quiz-data.json`은 생성물이라 gitignore됨.
- `.github/workflows/deploy.yml` — main 푸시 시 GitHub Pages 자동 빌드·배포.
  로컬에는 Node가 없으므로 빌드 검증은 GitHub Actions로 한다.

## 카드 형식
frontmatter 필드:
- `title` — 질문 한 줄 요약 (카드의 고유 식별자 역할)
- `tags` — 첫 항목은 대분류(알고리즘, 자료구조, 운영체제, 유니티, C#,
  디자인 패턴, 게임 수학·물리, 그래픽스 중 하나), 이후 세부 태그 2~4개
- `related` — 태그가 겹치는 수평 연관 카드의 제목 목록.
  대괄호 없는 문자열 배열만 허용: `related: ["카드 제목", "카드 제목"]`
- `parent` — 앞 질문의 답을 전제해야 성립하는 꼬리 질문일 때만 상위 카드
  제목 기재. parent가 있는 카드는 사이트 랜덤 퀴즈에서 제외된다.
- `date`, `result`(맞음/부분/틀림/모름), `status`(완성/스텁)

본문 섹션: 질문 / 핵심 답변 / 정리 / 꼬리 질문.
`[[위키링크]]`는 본문에서만 사용하고 frontmatter에는 쓰지 않는다.

## 정비 작업 ("정비해줘" 요청 시)
아래 순서로 수행한다. 카드의 본문 내용(질문, 답변, 정리)은 절대 수정하지
않는다. 정비는 메타데이터와 링크만 다룬다.

1. frontmatter 정규화 — 전체 카드를 스캔해 형식 오류를 고친다.
   특히 `related`에 `[[...]]` 대괄호가 섞여 있으면 문자열 배열로 변환.
2. related 상호 채우기 — 대분류 또는 세부 태그가 2개 이상 겹치는 카드끼리
   서로의 related에 추가한다. 기존 항목은 삭제하지 않고 추가만 한다.
   카드당 related는 5개를 넘기지 않는다(태그 겹침 수가 많은 순으로 선별).
3. 스텁 생성 — 본문의 `[[위키링크]]`가 가리키는 카드가 없으면
   `status: 스텁`, `result` 공란, 질문 섹션만 채운 스텁 카드를 생성한다.
   스텁의 질문은 링크 문구를 단독으로 성립하는 의문문으로 다듬는다.
4. index.md 재생성 — 전체 카드에서 자동 생성. 형식:
   `제목 | 대분류 | 세부태그 | result | status` (카드당 한 줄, 스텁 포함)
5. 변경 요약을 보고한 뒤 커밋·푸시한다. 커밋 메시지는
   `chore: 정비 YYYY-MM-DD (카드 N장, 스텁 +N, 링크 +N)` 형식.

## 사이트 (Quartz 4)
- `cards/`의 카드를 콘텐츠로 사용. 위키링크·태그 페이지·백링크는
  Quartz 기본 기능을 사용한다.
- 랜덤 퀴즈 페이지를 추가한다:
  - `parent`가 있는 카드와 `status: 스텁` 카드는 퀴즈 풀에서 제외
  - 무작위 카드의 "질문" 섹션만 먼저 표시하고, 버튼 클릭 시 해당 카드
    페이지로 이동해 답을 확인하는 방식
  - (선택) result가 틀림/모름인 카드만 출제하는 필터
- `parent`가 있는 카드는 페이지 상단에 상위 카드 링크를 표시한다.
- 배포는 GitHub Actions로 GitHub Pages에 자동 배포되도록 구성한다.

## 작업 원칙
- 카드 본문은 수정 금지. 발견한 내용 오류는 수정하지 말고 보고만 한다.
- 파일 삭제는 명시적 요청이 있을 때만.
- 푸시 전 변경 요약을 먼저 보여준다.
