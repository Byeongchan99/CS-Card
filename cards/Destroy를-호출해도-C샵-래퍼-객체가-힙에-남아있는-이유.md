## 질문
Destroy(go)를 호출하면 네이티브 객체는 파괴된다. 그런데 go라는 C# 래퍼 객체는 그 순간 힙에서 사라지는가? 사라지지 않는다면 무엇이 그것을 붙잡고 있는가?

## 핵심 답변
사라지지 않는다. Destroy는 네이티브 객체를 파괴할 뿐 C# 참조 그래프를 건드리지 않는다. C# 래퍼는 GC의 영역이고, GC는 루트에서 도달 가능한 참조가 하나라도 남아있는 객체를 살아있는 것으로 판단한다. 실행 중인 메서드의 지역변수, 다른 스크립트의 리스트·딕셔너리에 담긴 참조 등이 go를 가리키고 있으면 래퍼는 힙에 계속 살아있다. 모든 C# 참조가 끊겨야 다음 GC 때 수거된다.

## 정리

### 왜 그런가
"네이티브의 죽음"과 "C# 래퍼의 죽음"은 별개의 사건이다. Destroy는 전자만 처리한다. 후자는 순전히 GC의 도달성(reachability) 판정에 달려 있으므로, Destroy를 아무리 호출해도 코드가 래퍼를 붙들고 있는 한 래퍼는 수거되지 않는다.

### 내부 동작
- Destroy: 네이티브 객체 소멸 요청(실제 소멸은 프레임 끝).
- 래퍼 생존 여부: GC가 루트(지역변수·정적 필드·컬렉션 등)에서 도달 가능한지로 판정.
- 모든 참조가 끊긴 시점 이후의 GC 사이클에서 래퍼가 관리 힙에서 수거됨.

### 흔한 오해/함정
"참조를 끊었더라도 엔진이 프레임 끝까지 래퍼를 강제로 붙잡아 GC를 막는다"는 오해. 프레임 끝에 처리되는 건 네이티브 객체의 실제 소멸이지 C# 래퍼의 생존이 아니다. 모든 C# 참조를 끊었다면 래퍼는 다음 GC 때 정상적으로 수거 대상이 된다. 반대로 참조를 끊지 않으면 네이티브가 죽어도 래퍼는 계속 살아남아 fake null 상태를 만든다.
