## 질문
C#의 List&lt;T&gt;는 내부적으로 배열을 사용한다. 내부 배열이 가득 찬 상태에서 Add를 호출하면 어떤 일이 일어나고, 그런데도 Add의 시간 복잡도를 "상환 O(1)"이라고 말하는 이유는 무엇인가?

## 핵심 답변
내부 배열이 가득 차면 두 배 크기의 새 배열을 할당하고 기존 요소를 전부 복사한 뒤 교체하므로 그 순간은 O(N)이다. 하지만 N개를 채우는 동안 발생한 복사 비용의 총합이 1+2+4+...+N/2 ≈ N, 즉 등비수열이라 O(N)에 머문다. 총비용 약 2N을 N번의 연산에 나눠 부담시키면 연산당 상수이므로 상환 O(1)이다. 근거는 "리사이즈가 드물어서"가 아니라 "리사이즈 비용의 총합이 연산 횟수에 비례하는 수준으로 억제되어서"다.

## 정리
### 개념
상환 분석(amortized analysis)은 최악의 단일 연산이 아니라 연산 시퀀스 전체의 총비용을 연산 수로 나눠 평가한다. 개별 Add는 O(1)일 수도 O(N)일 수도 있지만, 어떤 N번의 Add 시퀀스도 총 O(N)을 넘지 않는다.

### 왜 그런가
```
캐파시티 변화:  4 → 8 → 16 → ... → N
복사 비용 합:   4 + 8 + ... + N/2  <  N   (등비수열)
쓰기 비용:      N
총비용:         약 2N  →  연산당 O(1)
```
등비수열은 마지막 항이 전체 합의 절반을 차지한다. "마지막 리사이즈 한 번"으로 전체를 근사해도 답이 맞는 건 이 성질 덕분이며, 논증 자체는 항상 총합으로 해야 안전하다.

### 흔한 오해/함정
"리사이즈가 가끔만 일어나니까 평균 O(1)"은 근거가 아니다. 고정 크기 증가 방식도 리사이즈는 가끔만 일어나지만 상환 O(N)이다. 빈도가 아니라 비용 총합의 스케일이 결정한다.
